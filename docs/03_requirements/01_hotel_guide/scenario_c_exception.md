## 二、核心研究问题 (RQs)

### RQ1: 异常识别与分类

> **机器人如何准确识别和分类用户遇到的异常情况？**

#### 子问题

| 编号 | 子问题 | 研究焦点 |
|------|--------|----------|
| RQ1a | 如何区分**问题严重程度**（轻微/中等/严重）？ | 严重程度 |
| RQ1b | 如何识别用户的**情绪状态**（平静/焦虑/愤怒）？ | 情绪识别 |
| RQ1c | 如何判断**紧急程度**（可等待/需立即处理）？ | 紧急程度 |

#### 异常类型分类

```
异常情况
    │
    ├── 导航类 ─────────────────────────────────────────────────┐
    │   • 迷路："我找不到我的房间了"                            │
    │   • 找不到设施："健身房在哪？"                            │
    │                                                          │
    ├── 服务类 ─────────────────────────────────────────────────┤
    │   • 投诉："房间空调坏了"                                  │
    │   • 请求："能帮我叫客房服务吗？"                          │
    │                                                          │
    └── 紧急类 ─────────────────────────────────────────────────┘
        • 身体不适："我感觉不舒服"
        • 安全问题："我的东西丢了"
```

### RQ2: 情绪安抚策略

> **机器人如何有效安抚用户的负面情绪？**

#### 子问题

| 编号 | 子问题 | 研究焦点 |
|------|--------|----------|
| RQ2a | **共情表达**（"我理解您的感受"）是否有效？ | 共情策略 |
| RQ2b | **道歉策略**（即使不是机器人的错）是否合适？ | 道歉策略 |
| RQ2c | **解决方案导向** vs **情感支持导向**哪个优先？ | 策略优先级 |

#### 预期发现

- 假设H2a：共情表达能显著降低用户愤怒程度
- 假设H2b：先情感支持后解决方案更受欢迎
- 假设H2c：机器人道歉会被接受但效果有限

### RQ3: 升级决策策略

> **机器人应该在什么情况下将问题升级给人工？**

#### 子问题

| 编号 | 子问题 | 研究焦点 |
|------|--------|----------|
| RQ3a | 用户对机器人**自主处理能力**的期望是什么？ | 能力期望 |
| RQ3b | **升级阈值**应该设在哪里？ | 升级标准 |
| RQ3c | 升级时如何**平滑交接**给人工？ | 交接方式 |
| RQ3d | 升级决策如何影响用户对机器人的**信任**？ | 信任影响 |

#### 升级决策模型

```
                    异常发生
                        │
                        ▼
              ┌─────────────────┐
              │  严重程度评估   │
              └────────┬────────┘
                       │
         ┌─────────────┼─────────────┐
         ▼             ▼             ▼
      轻微           中等           严重
         │             │             │
         ▼             ▼             ▼
    自主处理      尝试处理        立即升级
                  失败则升级
```

## 三、VR-WoZ 实验设计

### 3.1 实验设置

```
┌─────────────────────────────────────────────────────────────────┐
│                   VR 酒店场景                                   │
│                                                                 │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐                    │
│  │  大堂   │    │  走廊   │    │  房间   │                    │
│  │ (迷路点)│    │(投诉点) │    │(问题点) │                    │
│  └─────────┘    └─────────┘    └─────────┘                    │
│                                                                 │
│       👤 遇到问题的住客 ←──── 🤖 机器人                        │
│       (被试扮演)              (Wizard控制)                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 实验条件 (问题类型 × 处理策略)

| | **自主处理优先** | **快速升级** |
|---|---|---|
| **导航问题** | 机器人尝试解决 | 快速转人工 |
| **服务投诉** | 机器人尝试安抚+解决 | 快速转人工 |
| **紧急情况** | 机器人初步响应后升级 | 立即升级 |

### 3.3 测量指标

| 类型 | 指标 | 测量方法 |
|------|------|----------|
| **行为** | 解决时间、升级次数、交互轮数 | 系统日志 |
| **情绪** | 情绪变化（前后对比）、安抚效果 | 问卷 |
| **主观** | 满意度、信任度、问题解决感 | 问卷 (Likert) |
| **偏好** | 升级偏好、处理方式偏好 | 选择题 + 访谈 |

### 3.4 实验流程

```
1. 知情同意 + 背景问卷 (5min)
        ↓
2. 情绪诱导 (5min)
   - 通过情境描述诱导轻度负面情绪
        ↓
3. VR环境熟悉 (5min)
        ↓
4. 实验任务 (每条件8min × 6条件 = 48min)
   - 分多次session进行
   - 每次扮演遇到不同问题的住客
        ↓
5. 后测问卷 (10min)
        ↓
6. 半结构化访谈 (15min)
```
