> **创建日期**: 2026-02-13
> **目标**: 深挖"服务边界"和"模糊指令"话题，找出具体研究问题

---

## 一、两个核心话题

### 1.1 服务边界 (Service Boundary)

**核心问题**：机器人应该做到什么程度？在哪里停下来？

| 维度 | 举例 | 边界问题 |
|------|------|----------|
| **空间边界** | 卧室、浴室、衣柜 | 机器人能进哪些房间？ |
| **物品边界** | 私人物品、贵重物品 | 机器人能碰哪些东西？ |
| **信息边界** | 看到的、听到的 | 机器人能"知道"什么？ |
| **决策边界** | 主动行为 | 机器人能自己决定做什么？ |
| **时间边界** | 何时服务 | 机器人什么时候该出现/消失？ |

#### 潜在研究问题

1. 用户对机器人进入不同空间的接受度差异？
2. 隐私敏感度如何影响服务边界设定？
3. 信任度与服务边界的关系？
4. 服务边界应该由谁设定？（用户主动设定 vs 机器人学习）
5. 服务边界是否应该动态变化？（陌生→熟悉）

---

### 1.2 模糊指令 (Ambiguous Commands)

**核心问题**：当用户指令不明确时，机器人应该怎么做？

| 类型 | 举例 | 模糊点 |
|------|------|--------|
| **程度模糊** | "把房间收拾一下" | 收拾到什么程度？ |
| **范围模糊** | "帮我整理东西" | 整理哪些东西？ |
| **偏好模糊** | "弄点吃的" | 做什么？怎么做？ |
| **时间模糊** | "一会儿提醒我" | 多久算"一会儿"？ |
| **对象模糊** | "把那个拿过来" | 哪个？ |
| **条件模糊** | "如果有人来就..." | 什么情况算"有人来"？ |

#### 潜在研究问题

1. 机器人应该**追问**还是**猜测**？
2. 追问的方式：语音？展示选项？默认执行+确认？
3. 猜测的依据：上下文？历史偏好？常识？
4. 猜错了怎么办？如何优雅地修正？
5. 用户对"被追问"的容忍度？（太笨 vs 太烦）

---

## 二、两个话题的交叉

```
                    明确指令              模糊指令
                 ┌─────────────────┬─────────────────┐
    服务边界内   │ 简单执行        │ 需要澄清/猜测   │
                 │ (无研究价值)    │ (研究点B)       │
                 ├─────────────────┼─────────────────┤
    服务边界外   │ 需要确认边界    │ 🔥 双重挑战     │
                 │ (研究点A)       │ (研究点C)       │
                 └─────────────────┴─────────────────┘
```

**🔥 研究点C最有价值**：模糊指令 + 边界问题同时出现

举例：
- "帮我收拾一下卧室" → 能进卧室吗？+ 收拾到什么程度？
- "把我的东西整理好" → 能碰私人物品吗？+ 怎么算整理好？

## 三、家庭任务场景枚举

### 3.1 按空间分类

| 空间 | 常见任务 | 边界敏感度 | 模糊指令频率 |
|------|----------|------------|--------------|
| 客厅 | 打扫、整理、待客 | 低 | 中 |
| 厨房 | 备餐、清洁、储物 | 低 | 高（口味偏好） |
| 卧室 | 整理、清洁、叠衣 | **高** | 高 |
| 浴室 | 清洁、备品补充 | **高** | 低 |
| 书房 | 整理文件、打扫 | 中（信息隐私） | 高 |
| 衣帽间 | 整理衣物、搭配建议 | **高** | 高 |

### 3.2 按任务类型分类

| 类型 | 举例 | 特点 |
|------|------|------|
| **清洁类** | 扫地、擦窗、吸尘 | 边界清晰，模糊度低 |
| **整理类** | 收纳、归类、摆放 | 边界模糊，个人偏好强 |
| **准备类** | 备餐、铺床、备衣 | 需要了解偏好 |
| **照护类** | 照顾宠物、植物 | 决策权模糊 |
| **社交类** | 接待访客、传话 | 信息边界敏感 |
| **采购类** | 补充日用品、食材 | 决策权+财务边界 |

## 四、🎯 候选实验场景

### 场景A：整理卧室

```
用户："帮我收拾一下卧室"

机器人面临的挑战：
1. 能否进入卧室？（空间边界）
2. 收拾到什么程度？（程度模糊）
3. 能否打开衣柜/抽屉？（物品边界）
4. 床上的东西怎么处理？（决策边界）
5. 发现私密物品怎么办？（信息边界）
```

### 场景B：准备晚餐

```
用户："晚上弄点吃的"

机器人面临的挑战：
1. 做什么菜？（偏好模糊）
2. 做多少？（量的模糊）
3. 能否自己决定用什么食材？（决策边界）
4. 能否自己去采购？（财务+空间边界）
5. 口味怎么把握？（历史学习）
```

### 场景C：接待访客

```
用户不在家，有人按门铃

机器人面临的挑战：
1. 能否代主人开门？（决策边界）
2. 让访客进入哪个区域？（空间边界）
3. 能透露主人什么信息？（信息边界）
4. 访客要求带话/留东西怎么办？（决策边界）
```

## 六、🔥 关键文献深度分析

### 6.1 AmbResVLM (2025) - 最相关论文

**核心贡献**：
1. 提出**视觉-语言模型**解决机器人任务歧义
2. 两阶段推理：
   - 阶段1：任务对象定位 + 歧义分类
   - 阶段2：生成用户查询 + 解释用户回复
3. 实验结果：消歧后任务成功率从69.6%提升到97.1%

**与我们研究的关系**：
- ✅ 验证了"模糊指令"是真实问题
- ✅ 提供了技术解决方案框架
- ⚠️ 但他们聚焦**物体级歧义**（哪个杯子？），我们可以聚焦**行为级歧义**（收拾到什么程度？）

**研究空白**：
```
AmbResVLM 解决的问题          我们可以研究的问题
─────────────────────────    ─────────────────────────
"把蓝色杯子放到托盘上"        "帮我收拾一下"
  ↓                            ↓
哪个蓝色杯子？(物体歧义)      收拾到什么程度？(程度歧义)
                              收拾哪些东西？(范围歧义)
                              能进卧室吗？(边界歧义)
```

### 6.2 ACM 2025 VR隐私研究 - 方法论参考

**关键发现**：
- 使用VR验证家用机器人设计
- **卧室隐私担忧最高**
- VR方法在HRI研究中越来越流行

**与我们研究的关系**：
- ✅ 验证了VR-WoZ方法的可行性
- ✅ 验证了"卧室"是高敏感场景
- ✅ 可以借鉴他们的实验设计

## 七、🎯 研究定位更新

### 7.1 现有研究 vs 我们的研究

| 维度 | 现有研究 | 我们的研究空白 |
|------|----------|----------------|
| **歧义类型** | 物体级（哪个杯子？） | 🆕 行为级（收拾到什么程度？） |
| **边界类型** | 隐私担忧（被监视） | 🆕 服务边界（能做什么？） |
| **交互方式** | 单次指令→执行 | 🆕 模糊指令→协商→执行 |
| **场景** | 简单物体操作 | 🆕 复杂家务任务 |

### 7.2 我们的独特贡献点

```
现有研究                      我们的创新
─────────────────────────    ─────────────────────────
物体歧义消解                  行为歧义消解
  "哪个杯子？"                  "收拾到什么程度？"

隐私担忧研究                  服务边界协商
  "机器人会监视我吗？"          "机器人能进卧室吗？"

技术解决方案                  交互设计研究
  VLM + 对话                   用户偏好 + 信任建立
```

## 九、✅ 场景确认：整理卧室

**确认时间**: 2026-02-13

> 📄 **详细设计文档**: [scenario_a_bedroom.md](./scenario_a_bedroom.md)

### 场景概要

| 项目 | 内容 |
|------|------|
| **场景名称** | 整理卧室 (Tidying Bedroom) |
| **核心指令** | "帮我收拾一下卧室" |
| **研究焦点** | 行为级歧义消解 + 服务边界协商 |

### 核心研究问题

| RQ | 问题 |
|----|------|
| **RQ1** | 用户如何与机器人协商卧室**服务边界**？ |
| **RQ2** | 用户对"收拾"的**期望差异**有多大？ |
| **RQ3** | 什么样的**协商方式**最有效？ |

### 实验设计

- **设计**: 2×2 (指令歧义程度 × 边界协商有无)
- **方法**: VR-WoZ
- **测量**: 协商次数、满意度、信任度、期望匹配度
