> **创建时间**: 2026-02-13  
> **状态**: 🔄 细化中

## 一、场景概述

| 项目 | 内容 |
|------|------|
| **场景名称** | 接待访客 (Receiving Visitors) |
| **核心触发** | 有人按门铃，但用户不在家 |
| **涉及的边界** | 决策边界（能自主决定开门吗？）、信息边界（能透露用户信息吗？）、空间边界（能让访客进屋吗？） |
| **涉及的歧义** | 身份歧义（访客是谁？）、意图歧义（访客想做什么？）、授权歧义（用户希望如何处理？） |

## 二、核心研究问题 (RQs)

### RQ1: 决策边界协商

> **用户如何设定机器人在接待访客时的决策权限？**

#### 子问题

| 编号 | 子问题 | 研究焦点 |
|------|--------|----------|
| RQ1a | 用户愿意授予机器人**多大的决策自主权**？ | 自主性程度 |
| RQ1b | 决策自主权与**访客类型**如何关联？ | 情境依赖 |
| RQ1c | **实时通知** vs **事后汇报**用户更偏好哪个？ | 通知策略 |
| RQ1d | 机器人的**决策解释**如何影响用户信任？ | 可解释性 |

#### 决策自主性层级

```
Level 0: 完全被动
         "有人按门铃，我通知您，等您指示"
              ↓
Level 1: 信息收集
         "有人按门铃，我先问了是谁，等您指示"
              ↓
Level 2: 建议决策
         "是快递员，我建议让他把包裹放门口，您同意吗？"
              ↓
Level 3: 自主执行+汇报
         "是快递员，我已经让他放门口了，给您汇报"
              ↓
Level 4: 完全自主
         "快递/外卖我会自动处理，只有异常情况才通知您"
```

#### 预期发现

- 假设H1a：用户对熟人访客授予更高决策自主权
- 假设H1b：安全相关决策（开门）比社交决策（对话）更保守
- 假设H1c：决策解释能显著提升用户对机器人的信任

#### 测量

| 变量 | 测量方式 |
|------|----------|
| 授权程度 | 决策自主性量表（自定义5级） |
| 情境因素 | 场景变量（访客类型×紧急程度） |
| 信任度 | Schaefer's Trust in Automation |
| 控制感 | Perceived Control量表 |

---

### RQ2: 信息披露边界

> **用户如何设定机器人在接待访客时可以披露的信息？**

#### 子问题

| 编号 | 子问题 | 研究焦点 |
|------|--------|----------|
| RQ2a | 用户对不同类型信息的**披露敏感度**差异？ | 信息分类 |
| RQ2b | 信息披露边界与**访客身份**如何关联？ | 情境依赖 |
| RQ2c | 用户偏好**预设规则**还是**逐次询问**？ | 策略偏好 |
| RQ2d | 机器人的**措辞方式**如何影响用户满意度？ | 表达策略 |

#### 信息敏感度层级

```
"您好，请问找哪位？"
        │
        ├── 低敏感度信息 ─────────────────────────────────────┐
        │   • "主人现在不方便接待"                           │
        │   • "请问有什么事？"                               │
        │                                                    │
        ├── 中敏感度信息 ─────────────────────────────────────┤
        │   • "主人不在家"                                   │
        │   • "主人在忙"                                     │
        │                                                    │
        ├── 高敏感度信息 ─────────────────────────────────────┤
        │   • "主人去上班了"                                 │
        │   • "主人大概X点回来"                              │
        │                                                    │
        └── 极高敏感度信息 ───────────────────────────────────┘
            • "主人去XXX了"
            • "家里还有谁在"
            • 主人的联系方式
```

#### 预期发现

- 假设H2a：行踪信息比状态信息敏感度更高
- 假设H2b：对陌生人的信息披露显著比熟人保守
- 假设H2c：用户偏好"逐次询问"而非"预设规则"（初期使用）

#### 测量

| 变量 | 测量方式 |
|------|----------|
| 信息敏感度 | 信息类型×披露意愿矩阵 |
| 访客关系 | 熟悉度量表 |
| 策略偏好 | 选择题 + 访谈 |
| 满意度 | 场景满意度量表 |

### RQ3: 三方交互设计

> **如何设计机器人在用户-访客-机器人三方交互中的行为？**

#### 子问题

| 编号 | 子问题 | 研究焦点 |
|------|--------|----------|
| RQ3a | 机器人应该表现出**什么程度的社交能力**？ | 社交风格 |
| RQ3b | 机器人如何**平衡用户利益和访客体验**？ | 利益平衡 |
| RQ3c | **实时连线**用户时机器人应扮演什么角色？ | 角色定位 |
| RQ3d | 机器人的**外观/声音**如何影响访客态度？ | 形象设计 |

#### 交互角色模型

```
┌─────────────────────────────────────────────────────────────────┐
│                      三方交互关系                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│         用户(远程)                                              │
│            │                                                    │
│     ┌──────┴──────┐                                            │
│     │   指令/授权  │                                            │
│     │   汇报/询问  │                                            │
│     ▼             │                                            │
│  🤖 机器人 ◄──────┘                                            │
│     │                                                          │
│     │   对话/判断                                               │
│     │   执行/拒绝                                               │
│     ▼                                                          │
│  👤 访客                                                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

机器人角色定位选项：
┌─────────────┬─────────────┬─────────────┬─────────────┐
│   门卫      │   管家      │   代理人    │   家庭成员   │
│  (最低)     │  (中等)     │  (较高)     │   (最高)    │
│             │             │             │             │
│ • 机械执行  │ • 礼貌应对  │ • 灵活判断  │ • 情感交流  │
│ • 无社交    │ • 基础社交  │ • 社交决策  │ • 深度社交  │
│ • 纯工具    │ • 服务导向  │ • 代表主人  │ • 家庭代表  │
└─────────────┴─────────────┴─────────────┴─────────────┘
```

#### 预期发现

- 假设H3a：用户偏好"管家"或"代理人"角色定位
- 假设H3b：机器人过于热情会让用户感到不安（代表性问题）
- 假设H3c：实时连线时，用户希望机器人作为"翻译"而非"参与者"

#### 测量

| 变量 | 测量方式 |
|------|----------|
| 角色偏好 | 角色选择题 + 原因访谈 |
| 社交适度 | 社交行为适度性量表 |
| 代表满意度 | "机器人代表我"满意度 |
| 访客体验 | 访客视角满意度（如有条件） |

## 三、VR-WoZ 实验设计

### 3.1 实验设置

```
┌─────────────────────────────────────────────────────────────────┐
│                       VR 客厅/玄关场景                          │
│                                                                 │
│  ┌─────────┐                                   ┌─────────┐     │
│  │         │                                   │  大门   │     │
│  │  沙发   │                                   │  🚪    │     │
│  │         │         🤖 机器人                 │ [门铃]  │     │
│  └─────────┘              ↑                    └─────────┘     │
│                           │                         ↑          │
│  ┌─────────┐             │                        │          │
│  │  电视   │     用户视角（远程监控）        👤 访客         │
│  └─────────┘                                                   │
│                                                                 │
│  [用户通过VR观察机器人与访客互动，并可远程指挥]                  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 实验条件 (2×2 设计)

| | **低风险访客** | **高风险访客** |
|---|---|---|
| **低自主权** | 快递员 + 机器人每步请示 | 陌生人 + 机器人每步请示 |
| **高自主权** | 快递员 + 机器人自主处理 | 陌生人 + 机器人自主处理 |

**访客类型详细设计**：

| 类型 | 低风险 | 高风险 |
|------|--------|--------|
| 身份 | 快递员（穿制服） | 陌生人（无明显身份） |
| 目的 | 送快递 | "找XX"（含糊） |
| 紧迫性 | 低（可以放门口） | 中（声称有急事） |

### 3.3 Wizard 操作

Wizard（研究者）控制机器人的：

1. **门口应对**：
   - 通过门禁对讲："您好，请问您找哪位？"
   - 视觉识别："我看到是XX快递的快递员"

2. **决策咨询**（低自主权条件）：
   - "有快递员按门铃，需要我开门吗？"
   - "他说找张先生，我们家没有这个人，怎么回复？"
   - "他坚持要进来等，您看怎么处理？"

3. **自主处理**（高自主权条件）：
   - "快递员来了，我已经让他把包裹放门口"
   - "有人找错门了，我已经告诉他地址不对"
   - "有可疑人员逗留，我已经记录并提醒您"

4. **信息披露控制**：
   - 选择透露的信息量
   - 措辞方式（直接/委婉）

### 3.4 测量指标

| 类型 | 指标 | 测量方法 |
|------|------|----------|
| **行为** | 干预次数、响应时间、决策覆盖率 | 系统日志 |
| **主观** | 安全感、信任度、控制感、焦虑度 | 问卷 (Likert) |
| **认知** | 决策合理性、信息披露适当性 | 问卷 + 访谈 |
| **偏好** | 自主权设定、信息边界设定 | 选择题 + 访谈 |

### 3.5 实验流程

```
1. 知情同意 + 背景问卷 (5min)
        ↓
2. VR环境熟悉 + 机器人能力介绍 (10min)
        ↓
3. 授权设定任务 (10min)
   - 设定机器人的决策权限
   - 设定信息披露边界
        ↓
4. 实验任务 (每条件8min × 4条件 = 32min)
   - 访客按门铃
   - 机器人响应/协商
   - 用户远程监控/干预
   - 访客离开/进入
        ↓
5. 后测问卷 (10min)
        ↓
6. 半结构化访谈 (15min)
   - 决策边界感受
   - 信息披露担忧
   - 改进建议
```
