## 二、核心研究问题 (RQs)

### RQ1: 身份验证与信任建立

> **机器人如何向司机证明自己的"乘客"身份？**

#### 子问题

| 编号 | 子问题 | 研究焦点 |
|------|--------|----------|
| RQ1a | 司机如何**确认**这是正确的机器人乘客？ | 身份确认 |
| RQ1b | **二维码/声音/APP**哪种验证方式最有效？ | 验证方式 |
| RQ1c | 司机对机器人乘客的**初始信任度**如何？ | 初始信任 |
| RQ1d | 如何处理**冒充/错误**的情况？ | 安全机制 |

#### 身份验证流程

```
主人下单                机器人等车              司机到达              验证身份
    │                      │                      │                    │
    ▼                      ▼                      ▼                    ▼
┌─────────┐          ┌─────────┐          ┌─────────┐          ┌─────────┐
│ APP下单 │          │ 在指定  │          │ 司机   │          │ ？？？  │
│ 注明是  │ ──────▶ │ 地点等  │ ──────▶ │ 到达   │ ──────▶ │ 如何    │
│ 机器人  │          │ 候      │          │        │          │ 验证？  │
└─────────┘          └─────────┘          └─────────┘          └─────────┘
```

### RQ2: 机器人-司机交互

> **机器人应该如何与司机沟通？**

#### 子问题

| 编号 | 子问题 | 研究焦点 |
|------|--------|----------|
| RQ2a | 机器人需要**主动交流**还是**保持沉默**？ | 交流程度 |
| RQ2b | 司机对机器人的**期望**是什么？ | 司机期望 |
| RQ2c | 遇到**路线变更**等情况如何处理？ | 异常沟通 |
| RQ2d | 机器人应该表现得像**乘客**还是**货物**？ | 角色定位 |

#### 角色定位选项

```
货物 ◄────────────────────────────────────────────► 乘客

│                        │                        │
▼                        ▼                        ▼
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 静默模式    │    │ 基本交互    │    │ 完全乘客    │
│             │    │             │    │             │
│ • 不说话    │    │ • 必要沟通  │    │ • 主动交流  │
│ • 被动运输  │    │ • 回应询问  │    │ • 礼貌互动  │
│ • 如快递    │    │ • 基本礼貌  │    │ • 如人类    │
└─────────────┘    └─────────────┘    └─────────────┘
```

### RQ3: 远程监控与主人参与

> **主人应该在多大程度上参与机器人的乘车过程？**

#### 子问题

| 编号 | 子问题 | 研究焦点 |
|------|--------|----------|
| RQ3a | 主人需要**实时监控**吗？ | 监控程度 |
| RQ3b | 出现问题时**谁来决策**？ | 决策权 |
| RQ3c | 远程监控如何影响**主人焦虑度**？ | 心理影响 |
| RQ3d | **主人-司机直接通话**是否必要？ | 沟通渠道 |

## 三、VR-WoZ 实验设计

### 3.1 实验设置

```
┌─────────────────────────────────────────────────────────────────┐
│                   VR 网约车场景                                  │
│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                        街边                                │ │
│  │                                                           │ │
│  │      🤖 机器人 ────────▶  🚗 网约车                      │ │
│  │      (Wizard控制)         (到达)                          │ │
│  │                                                           │ │
│  │      👤 司机（NPC或Wizard）                               │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  被试视角：                                                     │
│  ┌───────────────┐                                             │
│  │ 📱 手机APP    │  被试扮演"主人"，远程监控机器人乘车        │
│  │ 实时监控画面  │                                             │
│  └───────────────┘                                             │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 实验条件 (2×2 设计)

| | **基本交互模式** | **完全乘客模式** |
|---|---|---|
| **无远程监控** | 机器人基本交互+主人无监控 | 机器人完全乘客+主人无监控 |
| **有远程监控** | 机器人基本交互+主人实时监控 | 机器人完全乘客+主人实时监控 |

### 3.3 测量指标

| 类型 | 指标 | 测量方法 |
|------|------|----------|
| **行为** | 沟通次数、干预次数、完成时间 | 系统日志 |
| **主观** | 焦虑度、信任度、满意度 | 问卷 (Likert) |
| **偏好** | 交互模式偏好、监控程度偏好 | 选择题 + 访谈 |
