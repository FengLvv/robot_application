## 二、核心研究问题 (RQs)

### RQ1: 分离与情感处理

> **用户与机器人分离时有什么情感需求？**

#### 子问题

| 编号 | 子问题 | 研究焦点 |
|------|--------|----------|
| RQ1a | 用户会对机器人送修感到**焦虑/不安**吗？ | 分离焦虑 |
| RQ1b | 需要有**告别仪式**吗？ | 仪式设计 |
| RQ1c | 用户希望机器人**知道**自己要被送修吗？ | 信息告知 |
| RQ1d | **拟人化程度**高的用户焦虑是否更强？ | 个体差异 |

#### 分离焦虑模型

```
用户与机器人的关系
        │
        ├── 工具关系 ──────────────────▶ 低焦虑
        │   "只是个机器"                   • 关心功能恢复
        │                                   • 无情感负担
        │
        ├── 伙伴关系 ──────────────────▶ 中焦虑
        │   "像个宠物"                     • 担心"它"会不会不舒服
        │                                   • 想了解维修进度
        │
        └── 家人关系 ──────────────────▶ 高焦虑
            "像家庭成员"                   • "它生病了我很难过"
                                           • 需要情感支持
```

### RQ2: 隐私与数据安全

> **用户对机器人维修过程中的数据隐私有什么担忧？**

#### 子问题

| 编号 | 子问题 | 研究焦点 |
|------|--------|----------|
| RQ2a | 用户担心**哪些数据**被泄露？ | 隐私类型 |
| RQ2b | 是否需要在送修前**清除数据**？ | 数据处理 |
| RQ2c | 用户是否信任维修人员**不会查看**数据？ | 信任程度 |
| RQ2d | **记忆/对话历史**是否应该保留？ | 连续性需求 |

#### 隐私担忧分类

```
隐私类型                          担忧程度（预期）
─────────────────────────────────────────────────────
家庭布局/活动模式                 ████████░░ 高
对话记录/指令历史                 ████████░░ 高
个人偏好/习惯数据                 ██████░░░░ 中
面部/声纹数据                     ██████████ 极高
情感交互记录                       ████████░░ 高
```

### RQ3: 替代与重逢

> **维修期间和维修后应该如何设计体验？**

#### 子问题

| 编号 | 子问题 | 研究焦点 |
|------|--------|----------|
| RQ3a | 是否需要提供**替代机器人**？ | 替代方案 |
| RQ3b | 替代品需要有**相同的记忆**吗？ | 连续性 |
| RQ3c | 维修期间需要**进度更新**吗？ | 沟通需求 |
| RQ3d | 重逢时需要**特别设计**吗？ | 重逢体验 |

### 3.2 实验条件 (2×2 设计)

| | **简单交接** | **仪式化交接** |
|---|---|---|
| **无进度更新** | 简单交接+无更新 | 告别仪式+无更新 |
| **有进度更新** | 简单交接+实时更新 | 告别仪式+实时更新 |

### 3.3 测量指标

| 类型 | 指标 | 测量方法 |
|------|------|----------|
| **情感** | 分离焦虑、等待焦虑、重逢喜悦 | 问卷 (STAI改编) |
| **主观** | 满意度、信任度、隐私担忧 | 问卷 (Likert) |
| **偏好** | 交接方式偏好、沟通频率偏好 | 选择题 + 访谈 |
