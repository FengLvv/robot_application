# 人形机器人应用场景头脑风暴

## 背景与目的

### 当前卡点

1. **技术限制**：初步开发基于人形机器人，采用CV2识别真人动作来操作VR中的机器人
2. **应用前景不明**：人形机器人的具体应用场景尚不清晰
3. **研究需求**：必须找到一个未来人形机器人的场景，且该场景**非常有必要在落地之前进行交互实验**

### 场景筛选标准

一个好的实验场景应满足：
- [ ] 人形机器人是该场景的合理选择（而非其他形态机器人）
- [ ] 交互设计对用户体验有显著影响
- [ ] 在产品落地前进行交互验证是必要的
- [ ] 可以在VR环境中有效模拟
- [ ] 具有研究价值和实际应用前景

### 实验流程

1. **准备阶段**
   - 实验员A准备实验脚本和话题
   - 实验员B校准动作捕捉系统
   - 被试C佩戴VR设备，进入虚拟环境

2. **实验阶段**
   - 实验员A通过打字/语音控制机器人对话内容
   - 实验员B通过肢体动作控制机器人动作
   - 被试C在VR中与机器人进行自然交互

3. **数据收集**
   - 交互行为记录
   - 被试主观反馈
   - 交互效果评估

### 场景可验证性检查清单

每个场景必须满足以下条件才能纳入实验：

- [ ] **对话可控**：实验员A可以通过打字/语音控制机器人说什么
- [ ] **动作可控**：实验员B可以通过肢体动作控制机器人做什么
- [ ] **VR可呈现**：场景可以在VR环境中有效呈现
- [ ] **交互可观察**：被试的交互行为可以被记录和分析
- [ ] **效果可评估**：交互效果可以通过问卷/访谈等方式评估

### 筛选标准
- ✅ 人形机器人必要性（非轮式/四足可替代）
- ✅ 交互验证必要性（落地前需要测试）
- ✅ VR实验可行性（可在VR中有效呈现）
- ✅ 研究价值（可发表/应用）

### 排除的场景
- ❌ 停车场找车（人形必要性低，轮式更高效）
- ❌ 宠物般陪伴（四足机器人更合适）
- ❌ 考试监考（摄像头更不打扰）
- ❌ 地铁站引导（VR环境构建过于复杂）
