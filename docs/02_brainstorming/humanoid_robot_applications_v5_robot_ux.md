# V5: 机器人的用户体验 (Robot UX)

> **创建日期**: 2026-02-13
> **核心视角**: 假设未来人人都有机器人，人类世界需要如何适配？
> **关键转变**: 不是改机器人适应世界，而是改世界适应机器人

| 研究 | 内容 |
|------|------|
| **论文** | "Enhancing Robot Inclusivity in the Built Environment" (2024) |
| **方法** | 使用数字孪生技术评估建筑环境的机器人友好性 |
| **核心概念** | Robot-Inclusive Design（机器人包容性设计） |
| **三阶段方法** | 文档化 → 数字化 → 设计分析 |
| **发现** | 机器人包容性 ≠ 机器人效率，但包容性环境促进更好的可达性 |

| 概念 | 说明 | 来源 |

### 💡 关键洞察

1. **这不是新概念** - 学术界已有"Robot-Inclusive Design"研究，但主要聚焦建筑/室内
2. **深圳在行动** - 政府层面已开始规划机器人友好型城市
3. **研究空白** - **车辆适配机器人**的研究几乎没有！
4. **方法论可借鉴** - SUTD的数字孪生方法可用于VR-WoZ实验

## 一、核心洞察

### 传统思路 vs 新思路

| 传统思路 | 新思路 |
|----------|--------|
| 机器人如何适应人类环境？ | **人类环境如何适应机器人？** |
| 改机器人 | **改基础设施** |
| 机器人是"用户" | 机器人也有"用户体验"问题 |
| 单个机器人的能力 | **机器人群体的普遍需求** |

### 类比：无障碍设计的演进

```
轮椅用户 → 无障碍坡道、电梯、宽门
盲人用户 → 盲道、语音提示、触觉标识
机器人用户 → ???（待探索）
```

**关键假设**：当机器人数量达到一定规模，改造基础设施比改造每个机器人更经济。

## 三、深入：机器人上车问题

### 3.1 为什么机器人需要上车？

| 场景 | 说明 |
|------|------|
| 跟随主人出行 | 家庭助手陪同购物、就医 |
| 跨场所服务 | 从家到办公室继续服务 |
| 维修/升级 | 送去服务中心 |
| 共享机器人 | 类似共享单车，需要调度 |

### 3.2 车辆改造方向

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **改造副驾** | 空间大、易交互 | 占用座位 | 私家车、专车 |
| **改造后排** | 不影响前排 | 空间受限 | 家庭出行 |
| **后备箱方案** | 不占乘客空间 | 无法交互、散热差 | 短途运输 |
| **车顶方案** | 完全不占内部空间 | 风阻、安全性 | 特殊场景 |
| **专用舱位** | 最优体验 | 需要全新车型设计 | 未来车型 |

### 3.3 车辆需要的功能

| 功能 | 说明 |
|------|------|
| **自动开门** | 机器人靠近时识别并开门 |
| **机器人识别** | 识别"我的机器人" vs 陌生机器人 |
| **固定装置** | 行驶中安全固定机器人 |
| **充电接口** | 标准化充电接口 |
| **散热系统** | 为机器人提供散热 |
| **通信接口** | 车机与机器人数据交换 |

## 四、研究价值分析

### 4.1 为什么这个方向有价值？

| 维度 | 分析 |
|------|------|
| **新颖性** | ⭐⭐⭐⭐⭐ 几乎没有人从"机器人UX"角度研究 |

| **学术性** | ⭐⭐⭐⭐ 可发展为新的研究领域 |

### 4.2 可能的研究问题

1. **机器人上车交互设计**
   - 车辆如何识别并响应机器人靠近？
   - 最佳的"上车流程"是什么？
   - 人-机器人-车 三方如何协调？

2. **机器人乘车体验**
   - 机器人在车内的最佳位置？
   - 行驶中机器人应该做什么？（休眠？继续服务？）
   - 如何处理机器人的散热/充电需求？

3. **机器人友好型基础设施**
   - 什么是"机器人无障碍设计"？
   - 如何制定机器人通行标准？

│  建筑内部     →  建筑入口    →  公共空间   →  车辆  │

└─────────────────────────────────────────────────────┘
```

**重大发现**：车辆-机器人交互是巨大的研究空白！

### 6.2 为什么车辆适配机器人是研究空白？

| 原因 | 分析 |
|------|------|
| **时间尺度** | 人形机器人2024年才开始大规模商用，汽车设计周期5-7年 |
| **责任主体不明** | 是车企负责？还是机器人厂商负责？ |
| **标准缺失** | 没有"机器人尺寸/接口"国际标准 |
| **优先级低** | 当前机器人主要在固定场所工作，出行需求不迫切 |

### 6.3 参考深圳"1+1+N"框架思考

深圳的思路是：**训练场地 + 示范区 + 多个子中心**

| 深圳做法 | 我们可以借鉴 |
|----------|-------------|
| 封闭训练 → 开放街道 | VR模拟 → 真实测试 |
| 政府主导标准 | 提出设计指南建议 |
| 场景化训练 | 聚焦特定场景深入 |

### 6.4 机器人上车场景的细分

```
                    机器人上车
                        │
        ┌───────────────┼───────────────┐
        ▼               ▼               ▼
    私家车场景       网约车场景       公共交通
        │               │               │
   ┌────┴────┐     ┌────┴────┐     ┌────┴────┐
   ▼         ▼     ▼         ▼     ▼         ▼

 一起出门   去维修    自己出行   乘公交    乘公交

### 6.5 最有研究价值的子场景

| 场景 | 新颖性 | 实用性 | VR可行 | 推荐度 |
|------|--------|--------|--------|--------|

## 七、🚗 聚焦场景：机器人上私家车

### 7.1 用户旅程图

```
阶段1: 出发准备
┌──────────────────────────────────────────────────┐
│  主人: "我们出门吧"                              │
│  机器人: 收到指令，检查电量，准备出门             │
│  痛点: 机器人如何知道要"上车"而不是"走路"？      │
└──────────────────────────────────────────────────┘
                        ▼
阶段2: 接近车辆
┌──────────────────────────────────────────────────┐
│  机器人: 跟随主人走向车辆                         │
│  车辆: 检测到主人和机器人靠近                     │
│  痛点: 车如何识别"主人的机器人"？                 │
│       车应该开哪个门？                           │
└──────────────────────────────────────────────────┘
                        ▼
阶段3: 上车动作
┌──────────────────────────────────────────────────┐
│  车辆: 为机器人打开门                            │
│  机器人: 进入车辆，找到位置                       │
│  痛点: 人形机器人如何"坐"进车里？                │
│       谁先上车？人还是机器人？                   │
└──────────────────────────────────────────────────┘
                        ▼
阶段4: 就位固定
┌──────────────────────────────────────────────────┐
│  机器人: 在指定位置坐好/站好                      │
│  车辆: 启动固定装置、连接充电/散热                │
│  痛点: 如何安全固定？如何防止急刹车摔倒？        │
└──────────────────────────────────────────────────┘
                        ▼
阶段5: 行驶中
┌──────────────────────────────────────────────────┐
│  机器人: 进入"乘车模式"                          │
│  主人: 可能需要机器人服务（导航、聊天等）         │
│  痛点: 机器人应该"休眠"还是"待机"？              │
│       如何处理散热问题？                         │
└──────────────────────────────────────────────────┘
                        ▼
阶段6: 下车
┌──────────────────────────────────────────────────┐
│  车辆: 到达目的地，解除固定                       │
│  机器人: 等待主人指令后下车                       │
│  痛点: 下车顺序？如何确保安全？                  │
└──────────────────────────────────────────────────┘
```

### 7.2 每个阶段的研究问题

| 阶段 | 核心研究问题 | VR实验设计思路 |
|------|-------------|---------------|
| **接近车辆** | 车辆如何识别机器人身份？开门时机？ | 测试不同开门距离/时机的用户感受 |
| **上车动作** | 人机上车顺序？机器人入车姿态？ | 测试不同顺序/姿态的自然度感知 |
| **就位固定** | 最佳座位位置？固定方式感知？ | 测试不同位置对乘客舒适度影响 |
| **行驶中** | 机器人状态？与乘客交互？ | 测试机器人不同状态的用户偏好 |

### 7.3 汽车设计建议框架

| 设计层面 | 具体建议 |
|----------|----------|
| **物理空间** | 预留机器人位置（尺寸标准）、调整座椅/固定装置 |
| **接口** | 机器人识别传感器、充电接口、通信协议 |
| **软件** | 机器人检测算法、自动开门逻辑、乘车模式切换 |
| **安全** | 碰撞保护、紧急情况处理、机器人故障处理 |

## 九、🎯 研究方向总结

### 9.1 三个可行的研究方向

| 方向 | 核心问题 | 产出 | 推荐度 |
|------|----------|------|--------|
| **🥇 机器人上私家车** | 车辆如何适配机器人？上车流程设计？ | 汽车设计指南 | ⭐⭐⭐⭐⭐ |
| **🥈 机器人乘电梯** | 机器人与人共乘电梯的交互规则？ | 电梯交互设计指南 | ⭐⭐⭐⭐ |
| **🥉 机器人通过门禁** | 机器人身份识别与授权机制？ | 门禁系统设计建议 | ⭐⭐⭐ |

### 9.2 为什么"机器人上私家车"是最佳方向？

1. **研究空白最大** - 几乎没有现有研究
2. **产业价值高** - 汽车是万亿级产业，机器人是千亿级产业
3. **时机恰当** - 汽车智能化+机器人普及双趋势交汇
4. **VR-WoZ完美匹配** - 汽车内部环境Unity资源丰富，交互流程明确
5. **用户明确** - 车主/乘客，被试好找

### 9.3 与现有学术概念的关系

```
Robot-Inclusive Design (SUTD等)
        │
        ├── Robot-Inclusive Buildings（建筑）← 已有研究
        │
        ├── Robot-Inclusive Public Spaces（公共空间）← 深圳在探索
        │
        └── Robot-Inclusive Vehicles（车辆）← 🆕 我们的创新点
```

### 10.2 一次电量周期 (4-8小时)

#### 机器人的"一次电量"生活

```
电量100% ─────────────────────────────────────────────> 电量0%
   │                                                      │
   ▼                                                      ▼
 满电出发                                              必须充电
   │                                                      │
   ├── 高强度工作 (2-3小时)                               │
   ├── 中等强度 (4-5小时)                                 │
   └── 待机/轻度 (6-8小时)                                │
```

#### 痛点与适配需求

| 痛点 | 场景 | 基础设施适配 |
|------|------|-------------|
| **电量焦虑** | 外出时电量不足 | 公共充电桩网络 |
| **充电位置** | 在哪里充电？ | 家庭充电位、商场充电区、车载充电 |

### 10.8 💡 关键洞察

#### 发现1：车辆适配贯穿多个周期

```
一天周期：跟主人出门上班 ──────────────┐
                                       │
一次保修期：送去维修中心 ──────────────┼──> 🚗 机器人上车
                                       │
一生周期：二手交易运输、报废回收运输 ──┘
```

**结论**：机器人上车不是单一场景，而是贯穿整个生命周期的基础需求！

#### 发现2：不同周期的研究成熟度不同

```
研究成熟度
    高 │  ████ 一次电量（充电技术成熟）
       │  ██   一天（智能家居有研究）
       │  █    一个月（维护服务少）
       │  ░    一次保修期（送修物流空白）
    低 │  ░    一生（全生命周期几乎没有）
       └─────────────────────────────────
```

#### 发现3：情感维度被忽视

- 机器人"生病"时用户的焦虑
- 机器人"老化"时用户的不舍
- 机器人"死亡"时用户的告别
- **这是HCI研究的独特切入点！**
