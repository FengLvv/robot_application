> **最后更新**: 2026-02-13

## 项目概述

**项目名称**: VR-WoZ Toolkit
**目标**: 构建Unity VR-WoZ实验包，帮助设计研究人员低成本验证人形机器人交互设计
**技术栈**: Unity + PICO Unity Integration SDK + Python API + CV2动作捕捉
**限制**: 仅适用于人形机器人（CV2识别真人动作控制VR机器人）

## 实验设计

### 实验角色分工
- **实验员A**: 主控，把握实验进度，主动打字/说话开启话题
- **实验员B**: 配合A，控制机器人进行动作
- **被试C**: 戴VR眼镜，与虚拟机器人交互

### 关键约束：被试可行性
被试C必须能够使用VR设备，因此：
- ✅ 年轻人/大学生：容易招募，VR适用性好
- ❌ 老年人：难以招募，VR适用性差，伦理审批复杂

**核心洞察**: 被试不一定是最终用户。可以用年轻人验证交互设计。

## 对话历程摘要

### 2026-02-13 对话内容

#### 1. V3 非主流场景探索
- **用户需求**：远离主流赛道，从蓝领工人日常出发寻找机器人介入点
- **安装Skills**：`sociologist-analyst`（社会学分析）、`ux-researcher`（用户研究）
- **分析群体**：建筑工人、环卫工人、餐饮后厨、夜间保安、农业采摘、殡葬行业、物业维修
- **V3 Top 1**：夜间保安巡逻（人形必要性高+交互空间大+被试易招）

#### 2. V4 支付能力版
- **用户洞察**：V3群体无法支付人形机器人费用，需关注中产以上群体
- **分析群体**：高净值家庭、双职工中产、独居白领、小型企业主、高端养老社区、私人诊所、科研实验室
- **核心逻辑**：受益者=买单者（闭环），采用Tesla高端先行策略
- **V4 Top 1**：高端家庭助手（支付能力+人形必要性+交互价值全满分）

#### 3. 综合排名（Final Ranking）
- **用户需求**：综合四个文档，打分出前五方向
- **评估模型**：六维模型（支付能力20%+人形必要性20%+交互价值20%+VR可行15%+被试可行15%+行业价值10%）
- **最终Top 5**：
  1. 🥇 高端家庭助手（98分）- 六维均衡，完美闭环
  2. 🥈 独居白领生活助手（87分）- 目标用户即被试
  3. 🥉 夜间保安巡逻（86分）- 人形必要性极高
  4. 双职工育儿辅助（83分）- 痛点真实但涉及儿童伦理
  5. 高端养老社区服务（83分）- B端采购但被试有挑战

#### 4. 关键洞察总结

**四个版本的演进逻辑**：
```
V1: 机器人能做什么？ → 技术视角
V2: 什么场景最火？ → 资本视角
V3: 谁在默默吃苦？ → 社会学视角
V4: 谁愿意买单？ → 商业视角
Final: 综合最优 → 六维平衡
```

**为什么高端家庭助手是第一**：
- 唯一一个六维都接近满分的场景
- 用户=买单者=可招募被试的代表（完美闭环）
- 1X NEO 2026预售，市场即将爆发
- 研究成果可直接对接商业化产品
